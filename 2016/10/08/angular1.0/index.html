Angular学习笔记

发表于 2016-10-08   |  

Angular.js

AngularJS应用组成如下：

MVC

  • view（视图），即HTML。
  • Model(模型), 当前视图中可用的数据
  • Controller(控制器), 即 JavaScript 函数，可以添加或修改属性

MVVM

  • Model—View—View—Model
  • 数据双向绑定

Angularjs 特点

  • 模块化
  • 开发便捷，快速（jq用1000行 现在只需100行）
  • 代码结构清晰，逻辑只需要关注数据变化
  • 适合构建大型单页面应用

controller 方法

页面的控制器

  • 对应dom的控制方法
  • controller主要控制模型变量
  • 指定了dom的对应的作用域
  • 指定 controller：Ng-controller

scope

scope 是模型。
scope 是一个 JavaScript对象，带有属性和方法，这些属性和方法可以在视图和控制器中使用。

  • 带$都是Angular自己提供的依赖注入服务（service）
  • 服务提供相应的方法
  • $scope对应隔离的数据模型，对应DOM
  • $scope变量的改变对应DOM改变

Ng-repeat

  • angular 提供的指令
  • dom元素上的一个标记
  • 告诉angular怎么将行为和元素绑定
  • 或者变换元素和它的子元素
  • 封装的一系统功能
  • 循环遍历对象
  • 数组的时候：Ng-repeat=”item in object”、$index
  • 对象的时候：Ng-repeat=”(name,value)in object”

Ng-click

  • 再点击的时候激活元素
  • Ng-click监听激活对象
  • Ng-class添加激活类
  • Ng-click=””
  • $event为事件对象

Ng-class添加类

  • Ng-class可执行js
  • Ng-class=”{‘a’: ture,’b’: false}”
  • Ng-class=”a?’active’:”

页面路由

  • 多页面应用加载单个页面
  • 依赖ngRoute.js和ngRoute模块
  • 引入相关资源和模块
  • 配置ngRouteProvider
  • 使用ng-view

service 获取数据

  • 写一个服务用与获取数据：使用$http请求资源
  • 使用服务
  • 编写模板
  • $http({}).success().error();
  • 使用$digest或者$apply更新数据

$Q 使用promise模式

  • $q.defer()
  • $q.promise
  • Defered.resolve(data);
  • Defered.reject(data);

编写一个指令

restrict

  • 限定指令的使用方式
    – ‘A’-only matches attribute name
    – ‘E’-only matches element name
    – ‘C’-only matches class name
  • 如 EA E EC EAC

SCOPE

  • 隔离作用域
    – $
    – =
    – @

link

  • 编译阶段执行
  • 四个参数: scope element attrs tabsCtrl ; Require

translude

  • Ng-translude

其他常用指令

  • Ng-show ng-if ng-option ng-select等

组件间通讯

  • $scope.$on
  • $scope.$emit
  • $scope.$broadcast
#angular
认识cookie与session的区别与应用

  • 站点概览
